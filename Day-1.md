--- Before Day-1 ---
I already knew basic computer operations.
web browsing, and how to install software on Windows. I had heard about Python, Git, GitHub, Linux, and VS Code, but I had very little practical experience using them.

## Day-1 Checklist

- [x ] I can open a Linux terminal and recognize the shell prompt (`user@machine:~$`)
- [ x] I can run `python --version`, `git --version`, and `uv --version` without errors
- [ x] I can open VS Code connected to WSL/Linux using `code .`
- [x ] I know what `~` means and can navigate to my home directory using `cd ~`
- [x ] I can create folders and files using `mkdir` and `touch`, and list them with `ls`
- [ x] I know the difference between `.xyz`, `./xyz`, and `../xyz`
- [x ] I know that in Linux, everything is a file
- [x ] I can navigate to the C and D drives within WSL
- [ x] I understand the difference between `>` (overwrite) and `>>` (append)
- [x ] I have a GitHub account and have created the `tds-bootcamp` repository

--- After Day-1 ---
I learned Linux basics and terminal navigation.
I learned how to set up a development environment using Python, Git, VS Code, and WSL/Linux. I understood basic Linux navigation, file creation, and the concept of the home directory. I also learned how GitHub repositories are created and managed.

--- Feedback (Suggestions for the TDS Team) ---
The material is beginner friendly.
The bootcamp provides a good introduction for beginners. More screenshots and troubleshooting examples for Windows users would make the setup process even easier.
